# The Avalanches Forum — Wayback Archive

Preservation effort for the defunct official phpBB fan forum at `theavalanches.com`.
The live forum is gone (the hosting backend was wiped; see investigation notes). This
project reconstructs what survives in the **Internet Archive Wayback Machine** by
enumerating every captured forum page and pulling the raw HTML — **entirely on GitHub
Actions**, committed back here as plain HTML for later parsing.

## Two boards

| Path | Era (captures) | Distinct topics | Distinct posts |
|------|----------------|-----------------|----------------|
| `/phpBB2/` (old board) | 2002–2009 | 526 (t≤1140) | 2,011 (p≤8020) |
| `/forum/` (newer board) | 2007–2020 | 213 (t≤2935) | 449 (p≤24241) |

Independent ID spaces → coverage is additive (~700+ distinct topics total).

## Layout

```
manifests/
  phpBB2_manifest.json   forum_manifest.json   # raw Wayback CDX (status-200 captures)
  fetch_targets.tsv      # deduped fetch list (one row per content-distinct capture)
  coverage_report.md     # generated by verify.py after a harvest
scraper/
  build_targets.py       # manifests -> fetch_targets.tsv  (dedup by normalized-url + length)
  harvest.py             # resumable, polite Wayback fetch (stdlib only)
  verify.py              # coverage + stub/empty-page report
archive/
  phpBB2/<slug>/<timestamp>.html       # e.g. phpBB2/t358/20070115xxxxxx.html
  forum/<slug>/<timestamp>.html        # slug = t<id> | p<id> | f<id> | u<id> | <page>[_s<start>]
.github/workflows/harvest.yml          # 8-shard harvest + single commit job
```

## Running the harvest

1. Actions tab → **harvest** → **Run workflow** (optionally tune the per-request delay).
2. 8 shards fetch their slice via the Wayback `id_` (raw, un-rewritten) endpoint, polite
   and rate-limited; a `collect` job merges artifacts, runs `verify.py`, and commits.
3. **Resumable** — re-run anytime; already-committed pages are skipped, so a throttled run
   just continues. Wayback occasionally rate-limits shared CI IPs; backoff + resume handle it.

Regenerate the target list after editing manifests:
```bash
python3 scraper/build_targets.py
```

## Fetch scope

`viewtopic` (content), `viewforum` (structure), `memberlist` + `profile` (user data).
Login/search/posting/privmsg are skipped (no content). ~7,800 content-distinct captures.

## Notes / ethics

Public fan-forum content, fetched read-only from a public archive for preservation.
Raw HTML is stored uncompressed so it can be parsed into a structured dataset
(JSON/SQL via e.g. `phpbb-salvage`) and, optionally, a static read-only site.
